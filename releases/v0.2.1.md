# v0.2.1 — Trend-level post briefs

**Date:** 2026-07-09

## Summary

Moves brief generation from the whole report to **individual trends**, matching the salon workflow: search trends → pick one → create a post.

## Changed

- **Create post** on each trend card instead of report-level **Generate brief**
- API accepts optional `trend_name` for single-trend post briefs
- Post brief title and UI copy reflect one-trend focus
- Brief page adapts layout for single-trend briefs

## API / UI

| Change | Detail |
|--------|--------|
| `POST /api/briefs/generate` | `{ "report_id", "trend_name" }` — primary path |
| Web route | `/briefs/generate/:reportId/:trendName` |
| Report page | Per-trend **Create post** buttons |

Full-report brief generation (omit `trend_name`) remains available for API consumers.

git pull origin main
