# v0.2.0 — Trend briefs & content ideas

**Date:** 2026-07-09

## Summary

Adds an end-to-end **trend brief** pipeline on top of saved research reports: score and rank trends, generate salon-ready briefs with evidence and “why now,” and produce per-trend content ideas (captions, hashtags, product tie-ins).

## Added

- **Backend** — `TrendBrief` models, DB tables, repository, and `briefs/service.py` (scoring, brief generation, idea regeneration)
- **API** — `POST /api/briefs/generate`, `GET /api/briefs/latest`, `GET /api/briefs/{id}`, `POST /api/ideas/generate`
- **Web UI** — Generate brief from a report, brief detail page, per-item content ideas page
- **Tests** — API coverage with `StubBriefLLM`

## Changed

- Report page includes **Generate brief** entry point
- `specs/nails-trending.md` documents brief APIs and marks social content as partially built

## API / UI

| Endpoint | Purpose |
|----------|---------|
| `POST /api/briefs/generate` | Build brief from `report_id` |
| `GET /api/briefs/{id}` | Load brief with ranked items |
| `POST /api/ideas/generate` | Regenerate ideas for one brief item |

Web routes: `/briefs/generate/:reportId`, `/briefs/:id`, `/ideas/:briefItemId`
