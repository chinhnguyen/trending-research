# v0.3.0 — Video posts, background media, unified post UI

**Date:** 2026-07-10

## Summary

Adds short-form **video** generation, **background media jobs** with live in-option progress, and a cleaner **post setup** flow. Removes the duplicate ideas page so briefs are the single place to view and extend a post.

## Added

- **Short video posts** — `post_format: video` with OpenAI Sora (`OPENAI_VIDEO_MODEL`, `WILLBE_MEDIA_MAX_VIDEOS_PER_POST`)
- **Background media jobs** — SQLite `media_generation_jobs`, async enrichment, resume on server restart
- **Progress API** — `GET /api/media-jobs/{id}`, `active_media_jobs` on brief responses
- **Web progress** — Per-option progress bar with auto-polling (no reload needed)
- **Post setup picker** — Platform (Instagram / TikTok) and type (text with images / video) on create and when adding options
- **Instagram share** — Browser flow: copy caption, download media, open Instagram composer

## Changed

- Caption/copy returns immediately; images and videos render in the background
- Create post page uses one unified setup panel instead of separate platform/format sections
- Removed duplicate `/ideas/:briefItemId` page (redirects to `/briefs/:id`)
- `POST /api/ideas/generate` adds another option to an existing post (same page as brief)

## API / UI

| Endpoint | Purpose |
|----------|---------|
| `POST /api/briefs/generate` | `{ report_id, trend_name, platform, post_format }` |
| `POST /api/ideas/generate` | Add another post option to a brief item |
| `GET /api/media-jobs/{id}` | Poll job status and progress |

Web routes: `/briefs/generate/:reportId/:trendName` → `/briefs/:id`

## Upgrade

```bash
git pull origin main
git checkout v0.3.0
# redeploy:
cd infra && AWS_PROFILE=willbe-trends-deploy pulumi up
```

Set `OPENAI_VIDEO_MODEL=sora-2` and confirm Sora API access for video generation.
